# 素材 URL 与格式要求

> 本文集中说明提交 GhostCut 任务前的通用素材约束，包括本地文件上传、视频格式、图片格式、URL 字符要求和批量数量限制。

## 通用规则

| 约束项 | 说明 |
| --- | --- |
| 本地文件 | 任务创建接口通常不直接接收本地路径。如果用户提供本地视频、图片或 SRT 文件，先按[文件上传](./10-file-upload.md)上传并获取临时 URL。 |
| URL 可访问性 | 任务中的视频、图片或字幕 URL 应能被 GhostCut 服务端直接访问。不要传本机路径、内网地址或需要登录态的临时页面地址。 |
| URL 字符 | 视频 URL 和图片 URL 不能包含中文字符。如果用户提供的 URL 含中文，应先提醒用户更换或转义为可直接访问的 URL。 |
| 示例素材 | 用户只是想体验流程且没有素材时，可按[示例素材](../sample-assets.md)提供演示视频或图片 URL。真实业务质量验证应换成用户自己的素材。 |

## 视频要求

普通视频处理任务通常通过 `urls` 传入待处理视频地址。提交创建任务前，应先检查以下约束：

| 约束项 | 说明 |
| --- | --- |
| 支持格式 | `mp4`、`avi`、`mov`、`mkv`、`mpg`、`m3u8`、`m4v`。 |
| 本地视频 | 本地视频必须先上传获取 URL，不能直接把本地路径传给创建任务接口。 |
| URL 字符 | 视频 URL 不能包含中文字符。 |
| 批量数量 | 普通 `urls` 批量提交时，单次请求最多不超过 20 个。具体功能如另有更严格限制，以功能文档为准。 |

适用场景：

- `/v-w-c/gateway/ve/work/free` 下的普通视频处理能力。
- 视频基础处理、去字幕、字幕压制、OCR/ASR、背景音乐分离、视频翻译配音等功能。

## 图片要求

图片处理任务通常通过 `downloadInfo` 传入图片 URL JSON 字符串。提交创建任务前，应先检查以下约束：

| 约束项 | 说明 |
| --- | --- |
| 支持格式 | `png`、`jpeg`、`jpg`、`bmp`、`webp`。 |
| 图片尺寸 | 不超过 `2000 * 2000 px`。 |
| 文件大小 | 不超过 `50MB`。 |
| 本地图片 | 本地图片必须先上传获取 URL，不能直接把本地路径传给创建任务接口。 |
| URL 字符 | 图片 URL 不能包含中文字符。 |

`downloadInfo` 示例：

```json
{
  "downloadInfo": "{\"url\":\"https://example.com/input.png\"}"
}
```

图片处理完整参数见[AI 图片处理](./81-image-processing.md)。

## SRT 和字幕素材

普通字幕压制和部分译制出海流程会使用 SRT 文件：

- 本地 SRT 文件先按[文件上传](./10-file-upload.md)上传。
- 字幕文件 URL 应能直接访问。
- 字幕样式、字体和压制参数见[字幕样式和字体配置补充](./26-subtitle-style-and-fonts.md)。
- 译制出海字幕素材管理见[译制出海字幕素材管理](./61-series-subtitle-materials.md)。

## 提交前检查清单

1. 用户给的是本地文件时，是否已经上传并拿到 URL。
2. URL 是否公网可访问，且不依赖登录态或本机环境。
3. URL 是否不包含中文字符。
4. 视频或图片格式是否在支持列表中。
5. 图片尺寸和大小是否满足限制。
6. 普通视频 `urls` 批量数量是否不超过 20。
7. 具体功能文档是否还有更严格的素材限制。
